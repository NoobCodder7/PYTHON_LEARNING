# ЁЯза Key Concepts

## What is Abstraction?

Abstraction is the process of **hiding implementation details** and showing only the required functionality to the user.

### Real-Life Examples

- ATM Machine
- Car
- Mobile Phone
- TV Remote
- Coffee Machine

In all these examples, users interact with simple buttons or controls without knowing how the internal system works.

---

## Abstract Class

An Abstract Class is a class that acts as a blueprint for other classes.

It cannot be instantiated (an object cannot be created directly from it).

Example:

```python
from abc import ABC

class Animal(ABC):
    pass
```

---

## Abstract Method

An Abstract Method is a method that has only a declaration and **no implementation**.

Every child class must implement all abstract methods.

Example:

```python
from abc import ABC, abstractmethod

class Animal(ABC):

    @abstractmethod
    def sound(self):
        pass
```

---

## ABC (Abstract Base Class)

`ABC` is provided by Python's `abc` module.

It is used to create abstract classes.

Import statement:

```python
from abc import ABC, abstractmethod
```

---

## @abstractmethod

The `@abstractmethod` decorator tells Python that the method **must be implemented** by every child class.

Example:

```python
class Dog(Animal):

    def sound(self):
        print("Woof")
```

---

# ЁЯУЦ Rules of Abstraction

- Import `ABC` and `abstractmethod` from the `abc` module.
- The parent class must inherit from `ABC`.
- Abstract methods must use the `@abstractmethod` decorator.
- Every child class must implement all abstract methods.
- Objects cannot be created directly from an abstract class.
